# evidence/index.md — Ara evidence-layer index

Per the Ara paradigm, this layer holds **raw outputs only**. Every claim in
`../logic/claims.md` resolves through this index to a specific JSON or
report file. No prose, no narrative, no interpretation lives here.

This skeleton points to existing artifact paths in the repository proper.
A subsequent compilation pass by `tools/ara_compile.py` will copy the
relevant files into a normalized `results/` tree under this directory and
emit per-claim provenance metadata.

## Note on regeneration

Several entries are tagged `(regenerate)`. The pre-Ara repo did not always
keep a stable JSON-summary pointer alongside its experiment scripts. The
upcoming `tools/ara_compile.py` pass will:

1. Run each `(regenerate)` experiment at scaled-down configuration
   (Ara Seal Level 3).
2. Emit a normalized JSON summary under `evidence/results/<claim>/...`.
3. Update this index with the canonical path.

Until that pass lands, claims tied to `(regenerate)` paths are
**provisionally substantiated** by the prose record in
`../../paper_outline.md` and `../../writeup_draft.md`. An Ara reviewer in
strict mode would mark these claims as needing fresh evidence; a relaxed
reviewer accepts the prose pointer.
