# Full-stack source map

Companion to [`asymptotic_floor_candidate_inventory.md`](asymptotic_floor_candidate_inventory.md) and the PR #110 public submission context.

This is a source map for paper and collaboration follow-up. It is not part of the PR #110 submission runtime, not a claim that every listed method has contest-scale evidence, and not a roadmap promise. The point is to make the research lineage, local surfaces, evidence grade, and next validation gate legible in one place.

Anchor: [`commaai/comma_video_compression_challenge#110`](https://github.com/commaai/comma_video_compression_challenge/pull/110).

---

## Status legend

| Status | Meaning |
|---|---|
| `EMPIRICAL` | Has an end-to-end measurement on a contest-relevant axis. The axis and hardware still matter. |
| `LANDED TOOLING` | Reusable local tooling exists and has local tests or operational use, but it is not by itself a score claim. |
| `SCAFFOLDED` | Code, runbook, or partial integration exists. It still needs contest-axis validation. |
| `DESIGN-ONLY` | Design memo or source-backed hypothesis exists. No promoted implementation yet. |
| `EXTERNAL REFERENCE` | External paper, repo, blog, or production system used as lineage only. |

Promotion rule: a candidate becomes paper-grade contest evidence only after a deterministic archive, byte/sha custody, runtime custody, component recomputation, and paired axis labeling where applicable. A source-map row is a research pointer, not a leaderboard claim.

## B. What this source map says about the next frontier

The tested frontier/medalist HNeRV-family archives do not leave deliverable rate-term gains for post-hoc general-purpose recompression of the actual `archive.zip` member bytes. That does not mean the compression problem is exhausted. It means the frontier-lowering work has to move at least one earlier or deeper layer:

- Change the emitted representation: NeRV/HNeRV variants, SIREN, C3, Cool-Chic, VQ/VAE-style symbols, semantic-conditioned renderers.
- Change the prior: MAE/VideoMAE, V-JEPA, DreamerV3, openpilot/commaVQ policy-world-model priors, comma10k/comma2k19 pretraining, geometry/pose priors, SAM3/Falcon dense masks.
- Change training: curriculum, frozen-teacher distillation, QAT, post-training adaptation, test-time optimization, per-pair treatment.
- Change allocation: foveation, wavelets, UNIWARD, water-filling, master-gradient per-pair and per-byte pricing.
- Change the entropy model or grammar: Balle/CompressAI hyperpriors, STC/arithmetic coding, deterministic packet compiler, side-information proofing.
- Change composition: stack multiple measured layers only after each one exposes a byte-closed interface and an interaction-risk contract.

For PR #110, the public body should stay compact. This document is the right place for the named source surface, because it can be long, caveated, and citation-heavy without distracting from the submission packet.

## C. Paper-grade evidence checklist

For each row that graduates from source-map item to paper result, record:

1. External source lineage: paper, repo, version/commit when relevant.
2. Local implementation surface: files, commit, and whether it is paper-faithful, inspired, or independently reimplemented.
3. Evidence grade: `EMPIRICAL`, `LANDED TOOLING`, `SCAFFOLDED`, `DESIGN-ONLY`, or `EXTERNAL REFERENCE`.
4. Archive custody: exact archive path, byte count, SHA-256, member list, member sizes, compression method.
5. Runtime custody: runtime tree, dependency closure, no hidden score-affecting sidecars, no uncharged external fetch.
6. Eval custody: axis tag, hardware, command, component distances, recomputed score, logs.
7. Comparison baseline: same runtime or matching source runtime; never inferred cross-axis equivalence.
8. Failure mode if negative: implementation falsification, paradigm falsification, harness mismatch, or unvalidated proxy.
