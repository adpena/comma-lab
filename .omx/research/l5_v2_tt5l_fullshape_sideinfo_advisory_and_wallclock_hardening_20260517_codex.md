Evidence grade: `[macOS-CPU advisory]`

## Summary

This landing closes one concrete TT5L evidence gap without claiming score:
current-code TT5L can emit a full 600-pair archive whose side-info channel is
nonzero for every pair. It remains non-promotional because the run used local
macOS CPU, skipped auth eval, and intentionally tripped a short wall-clock cap
before validation.

Two engineering bugs were found and fixed before preserving the artifact:

1. `--max-wall-clock-hours` was only checked between epochs. A 600-pair CPU
   advisory epoch could overrun the cap because the trainer spends most time
   inside the per-pair batch loop. The trainer now checks the cap before each
   batch and records the batch location in the abort message.
2. Fallback-before-validation emitted `best_val_lag: Infinity` in
   `provenance.json`. Provenance now writes strict JSON with `allow_nan=False`
   and converts non-finite floats to `null`.
