# Codex Findings: Contract-First Candidate Intake And Partner Gate Repair

- UTC: 2026-05-30T23:47:02Z
- Commit target: main
- Scope: candidate queue archive intakes, field-meta archive identity selection, source/embedded archive-bound candidate contracts, and partner PR110 trainer gate review.

## Finding

Two stale-field surfaces still let archive-bearing rows be interpreted outside
the shared `archive_bound_candidate_contract` contract:

- legacy candidate-queue rows with `candidate_archive_path` could remain
  archive-verified without emitting a reusable contract snapshot;
- field-meta dispatch selection could prefer flat `candidate_archive_*` or
  `archive` identity fields over a fresher source contract, allowing stale
  duplicate SHA/byte values to outrank the canonical custody surface.

The focused partner landing review also found a committed PR110 trainer helper
wire-in that passed its behavioral tests but failed `ruff` on unused imports,
unused `noqa`, import ordering, unused local state, and an unused test import.

## Fix

Candidate queue intake now auto-wraps path-carrying archive rows into an
embedded archive-bound candidate contract before archive verification. The
wrapper is intentionally narrow: it fires only for rows with real archive path
custody and it does not convert MLX/local/advisory rows or raw exact-ready bits
into dispatch authority.

Autogenerated embedded contracts remain false-authority for exact dispatch:
raw `ready_for_exact_eval_dispatch=true` is blocked unless the row carries a
source contract snapshot from the exact-ready bridge. Invalid truthy authority
fields now fail closed while still preserving observed archive SHA diagnostics.

Field-meta dispatch selection now resolves archive identity from
`source_archive_bound_candidate_contract` or embedded
`archive_bound_candidate_contract` first. Invalid or mismatched contracts become
the selected blocked proof, so stale flat duplicate fields cannot hide the
contract failure.

The PR110 trainer partner landing was mechanically repaired so the committed
Python surface is lint-clean without changing the tested helper semantics.

## Review
