## 4. The 4 candidate architectures (FULL spec per Council Revision #4)

### Candidate 1 — Multi-layer FiLM (depth=3, ~300K params)

**Source of recommendation**: Council Revision #4 verbatim "(1) Multi-layer FiLM (higher predictor
capacity than Z6-v1's single layer)" + Rao verbatim "Z6-v1 single-layer 75K-param FiLM likely
lacks representational capacity to express semantic ego-conditioning at the scorer's stride-2 stem"

**Architecture**:

```
upstream/videos/0.mkv (1200 frames @ 874x1164 RGB)
        |
        v (pyav decode + canonical tac.substrates._shared.trainer_skeleton.decode_real_pairs)
        |
   pair_tensor :: (600, 2, 3, 384, 512)
        |
        v [Encoder (same as Z6-v1)]
        |
   latent_pair :: (600, 2, 64, 24, 32)
        |
        v [POSENET (compress-only; not in archive)]
        |
   ego_motion :: (600, 8)  -- PoseNet projection (Z6-v1 baseline) OR alternative source (CC-5)
        |
   ┌────────────────────────────────────────────────────────┐
   │  PER-PAIR AUTOREGRESSIVE FILM PREDICTOR (depth=3)      │
   │                                                         │
   │  for t = 0 .. 599:                                      │
   │      h0 = latent_pair[t-1] if t > 0 else latent_init    │
   │                                                         │
   │      [Layer 1 FiLM]                                      │
   │      film1_gamma, film1_beta = MLP_ego_1(ego_motion[t]) │
   │      h1 = Conv(h0, kernel=3) * film1_gamma + film1_beta │
   │      h1 = ReLU(h1)                                      │
   │                                                         │
   │      [Layer 2 FiLM]                                      │
   │      film2_gamma, film2_beta = MLP_ego_2(ego_motion[t]) │
   │      h2 = Conv(h1, kernel=3) * film2_gamma + film2_beta │
   │      h2 = ReLU(h2)                                      │
   │                                                         │
   │      [Layer 3 FiLM]                                      │
   │      film3_gamma, film3_beta = MLP_ego_3(ego_motion[t]) │
   │      h3 = Conv(h2, kernel=3) * film3_gamma + film3_beta │
   │                                                         │
   │      latent_pair[t] = h3                                │
   │                                                         │
   │      residual[t] = latent_pair[t] - latent_pair[t-1]    │
   └────────────────────────────────────────────────────────┘
        |
        v [Quantize residual int8]
        |
   archive: encoder_state_dict_fp16_brotli +
            decoder_state_dict_fp16_brotli +
            predictor_state_dict_fp16_brotli (3 × FiLM layers + 3 × ego MLPs) +
            latent_init_int8 +
            residuals_int8 +
            ego_motion_int8_sidecar +
            meta_json
```

**Param count breakdown**:
- Encoder + decoder (shared with Z6-v1): ~75K combined
- FiLM Layer 1 (Conv 64×64 k=3 + ego MLP): ~36K + ~1K = ~37K
- FiLM Layer 2 (Conv 64×64 k=3 + ego MLP): ~37K
- FiLM Layer 3 (Conv 64×64 k=3 + ego MLP): ~37K
- Latent init: ~50K (Z6-v1 baseline)
- **Total: ~236K params (4.0× Z6-v1's 60K predictor portion; ~3.1× Z6-v1's 75K full)**
- **Archive size estimate**: ~150 KB (Z6-v1 ~95 KB + ~50 KB predictor overhead + ~5 KB meta growth)

**Bytes added (vs Z6-v1)**: ~+55 KB on archive size. Per Catalog #220 + #272 distinguishing-feature
contract: the multi-layer FiLM predictor IS the distinguishing primitive; the runtime overlay
consumes it via the autoregressive unroll; byte-mutation smoke MUST verify mutations on any FiLM
layer's weights produce measurable downstream frame changes.

**Cargo-cults unwound**: CC-1 (capacity) + CC-2 (single FiLM stem) + CC-6 (smoke-time disambiguator
wire-in preserved from Z6-v1 paired-control fix)

**Mathematical justification**:
Per Rao-Ballard 1999, hierarchical predictive coding requires N≥3 levels for canonical operation
(top-level latent + 2+ residual-prediction levels). The 3-layer FiLM approximates this hierarchy
in a single-spatial-resolution form: Layer 1 captures coarse motion (large-scale ego-aligned
transformations); Layer 2 captures mid-scale structure (per-region object motion); Layer 3 captures
fine-scale residuals (per-pixel adjustments). Each layer's FiLM gate is independently ego-modulated
so the predictor can express conditional motion-magnitude scaling at each scale.

Per Tishby IB framework: I(T;Y) increases with predictor capacity until a saturation point;
Z6-v1's 75K params likely fell below the saturation point on the contest scorer; ~300K params
empirically sits at the typical saturation range for similar conditional-image-prediction tasks
(per Perez 2017 FiLM ablations on CLEVR; sister of NeRV / HNeRV PR101 architecture sizes).

**Predicted ΔS band**: `[0.13, 0.17]` `[prediction; Dykstra-feasibility-validated;
HIGH VARIANCE pending paired CPU/CUDA empirical anchor]`. Slightly wider than Z6-v1's [0.13, 0.16]
to reflect capacity uncertainty: the upper bound shifts to 0.17 because increased predictor
capacity introduces overhead that may not be fully amortized at low-residual regimes.

**Dispatch cost estimate**:
- Smoke (Catalog #167): **$1.50** Modal T4 (3-layer unroll is 1.5× slower than Z6-v1; ~30 min
  wall-clock for 1-epoch + roundtrip)
- Disambiguator at SAME archive bytes (Council Revision #2): **$1** Modal T4 + **$0.50** GHA
  Linux x86_64
- Full dispatch (Council Revision #3 PROCEED branch): **$15** Modal T4 (300 epochs × 1.5×
  predictor cost = ~45 min wall-clock per epoch; +50% over Z6-v1's $10)
- Paired CPU/CUDA: **$1.50** (same as Z6-v1)
- **TOTAL Candidate 1 path**: **$19.50** (vs Z6-v1 Path A $12.50; +$7 for capacity unwind)

**Engineering risk**: LOW-MEDIUM. The trainer skeleton + scorer-loss helper + archive grammar +
autoregressive unroll + EMA + eval_roundtrip wiring all reuse Z6-v1's canonical patterns. The ONLY
substrate-engineering edit is: replace single-layer FiLM predictor with 3-layer FiLM stack. Sister
LOC budget estimate: ~+80 LOC (3× the single-layer's ~25 LOC predictor block + 3× ego MLP
parameter init). Inflate runtime budget: ~+15 LOC for the deeper unroll; well within the 200 LOC
HNeRV parity L4 budget.

### Candidate 2 — Z7 LSTM recurrent predictor (~250K params)

**Source of recommendation**: Council Revision #4 verbatim "(2) Z7 LSTM recurrent predictor (sister
architecture at higher tier; Catalog #311 enforced)" + Rao verbatim "Z7 multi-layer or recurrent
predictor warranted" + Z6/Z7/Z8 design memo §3 Z7 spec (already designed; the design memo provides
the full spec).

**Architecture**:

```
ego_motion :: (600, 8)
        |
   ┌────────────────────────────────────────────────────────────┐
   │  PER-PAIR LSTM RECURRENT PREDICTOR                          │
   │                                                              │
   │  h_0 = LSTM_init_state                                       │
   │  for t = 0 .. 599:                                           │
   │      x_t = concat(latent_pair[t-1], ego_motion[t])           │
   │      h_t, c_t = LSTM(x_t, h_{t-1}, c_{t-1})                 │
   │      latent_pair[t] = Conv(h_t, kernel=1)  # project hidden  │
   │      residual[t] = latent_pair[t] - latent_pair[t-1]         │
   └────────────────────────────────────────────────────────────┘
```

**Param count**: hidden_dim=128 LSTM (~80K) + projection conv (~16K) + encoder/decoder (~75K) +
latent_init (~50K) = ~221K

**Bytes added (vs Z6-v1)**: ~+50 KB on archive size

**Cargo-cults unwound**: CC-1 (capacity via LSTM hidden state) + CC-2 (recurrent state IS sequential
hierarchy beyond single-frame conditioning) + temporal modeling of slow motion patterns across many
frames (something single-layer FiLM CANNOT do)

**Mathematical justification**: Per Hafner DreamerV3 + Rao predictive coding: recurrent state-space
models capture temporal coherence in driving video that per-pair FiLM CANNOT. The LSTM hidden state
accumulates information about long-range motion patterns (e.g. road curvature; constant velocity
sequences) that constant-conditioning predictor primitives miss. Per the Wyner-Ziv 1976 framing:
the LSTM's hidden state IS an implicit side-information channel between the encoder and decoder
when both run the same recurrence; the per-pair residual encodes only the unpredicted part.

**Predicted ΔS band**: `[0.11, 0.16]` `[prediction; Dykstra-feasibility-validated; potential class-
shift via temporal coherence; HIGH VARIANCE pending empirical anchor]`. Lower bound 0.11 reflects
potential class-shift via temporal coherence capturing patterns that no FiLM-only architecture
can express; upper bound 0.16 matches Z6-v1's because if LSTM does not improve over FiLM the
configuration degrades to Z6-v1 baseline.

**Dispatch cost estimate**:
- Smoke: **$2** Modal T4 (RNN backprop-through-time across 600 pairs is 2× slower than Z6-v1's
  feedforward unroll; ~40 min wall-clock for 1-epoch + roundtrip)
- Disambiguator: **$1** Modal T4 + **$0.50** GHA Linux x86_64
- Full dispatch: **$20** Modal T4 (300 epochs × 2× cost = ~60 min wall-clock per epoch)
- Paired CPU/CUDA: **$1.50** (same as Z6-v1)
- **TOTAL Candidate 2 path**: **$25** (vs Candidate 1 $19.50; +$5.50 for sequential dynamics
  unwind)

**Engineering risk**: MEDIUM. RNN backprop-through-time + LSTM forward/backward pass adds
implementation complexity vs Candidate 1's feedforward FiLM. The Z6/Z7/Z8 scoping design memo
already specified the Z7 LSTM architecture (Section 3 Z7 + Section 4.2), so the substrate-
engineering work has prior design depth. Sister LOC budget estimate: ~+150 LOC (LSTM module +
RNN unroll + hidden-state archive serialization). Inflate runtime budget: ~+40 LOC for LSTM
forward pass + recurrent state initialization; within budget.

### Candidate 3 — Comma2k19 pretraining (~75K params, Z6-v1 baseline + pretrained weights)

**Source of recommendation**: Council Revision #4 verbatim "(3) Comma2k19 pretraining stage (per

**Architecture**: SAME as Z6-v1 (single-layer FiLM, ~75K params), BUT initialized from a

**Pretraining pipeline**:

```
Stage 0 (pretraining; NEW for Candidate 3):
  Source: Comma2k19LocalCache (DP1 sister; ~20 hours of dashcam @ ~30 chunks)

  Epochs: 1-5 on full corpus (cost estimate: $3-15 cloud GPU pretraining ON TOP OF Path A budget)
  Checkpoint: predictor_state_dict_pretrained_comma2k19_<epoch>.pt

Stage 1 (Z6-v1 score-aware fine-tune; SAME as Z6-v1 Path A):
  Initialize: load predictor_state_dict_pretrained_comma2k19 (NEW; Candidate 3 distinguishing primitive)
```
