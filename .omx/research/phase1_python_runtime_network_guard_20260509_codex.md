# Phase 1 Python runtime network guard - 2026-05-09

Generated by: `codex`

## Finding

Read-only adversarial review found that `src/tac/phase1_packet_compiler.py`
blocked network/dependency-fetch tokens in `inflate.sh`, but `_scan_inflate_py`
did not apply the same `FORBIDDEN_NETWORK_TOKENS` list to `inflate.py`.

This left a hermeticity gap: a Phase 1 packet could hide `uv pip install`,
`pip install`, tokenized package-manager calls such as
`['uv', 'pip', 'install', ...]`, `curl`, `wget`, Python HTTP libraries, or URL
fetch literals in Python runtime code while passing the shell scan.

## Fix

`_scan_inflate_py()` now runs `_scan_text_for_forbidden(...)` with
`FORBIDDEN_NETWORK_TOKENS`, in addition to scorer-load and external-state
tokens. The forbidden token list now also includes common tokenized Python
package-manager and URL-fetch forms (`'uv', 'pip', 'install'`,
`urllib.request`, `requests.`, `httpx.`, `urlopen(`, `urlretrieve(`).

Focused regression added:

```bash
  src/tac/tests/test_build_phase1_packet_compiler.py::test_fail_closed_on_uv_runtime_dependency_fetch_in_sh \

```

Result: `3 passed`.

## Classification

Bug class: Phase 1 runtime hermeticity gap.

Impact: fixed before any Phase 1/T1 remote dispatch. No score claim changed.

Reactivation criterion: any future runtime dependency fetch mechanism must be
declared as a contest-forbidden packet blocker unless the contest runner ships
that dependency already and the runtime only imports it locally without network
or package-manager access.
