FILE 1: src/tac/stc_boundary_codec.py (~250-350 lines)
Contract:
  - detect_boundary_pixels(mask_class_ids: torch.Tensor[N,H,W] int, threshold_quantile: float = 0.95) -> torch.Tensor[N,H,W] bool
      Boundary detection from class-ID maps via Sobel-magnitude on argmax (no scorer needed at decode). Returns top-(1 - threshold_quantile) fraction of pixels by gradient magnitude.
  - encode_stc_syndrome(boundary_pixels: torch.Tensor[N,H,W] bool, class_ids: torch.Tensor[N,H,W] int, num_classes: int = 5) -> bytes
      Encodes ONLY the class IDs at boundary positions via arithmetic coding (using arithmetic_qint_codec primitives if available, otherwise inline a stable binary arithmetic coder with a 5-symbol model). Bitstream format: header (N, H, W, num_classes, boundary_count) + boundary bitmap (run-length encoded) + class-ID arithmetic stream for boundary positions only.
  - decode_stc_syndrome(payload: bytes, fill_class: int = 0) -> torch.Tensor[N,H,W] int
      Reverses encode. Non-boundary pixels filled with fill_class (operator can post-process by majority-class-fill from neighbors, but default is constant-fill). NOTE: for true lossless, fill_class should be derived per-pixel from the dominant class — simpler approach for v1: encode majority class per N x M tile (tile_size=32) as a small auxiliary table, fill non-boundary pixels from tile majority.
  - encode_mask_video_stc(masks: torch.Tensor[N,H,W] int, output_path: Path, num_classes: int = 5, threshold_quantile: float = 0.95) -> int (bytes_written)
  - decode_mask_video_stc(input_path: Path) -> torch.Tensor[N,H,W] int
  Top-level interface mirrors mask_codec.encode_masks_auto / decode_masks_auto signatures.

Critical:
  - Bitexact: encode then decode must yield IDENTICAL class IDs at boundary positions
  - For non-boundary positions, the recovered ID must match the tile-majority encoding (test with majority-class-only frames returns 100 percent recovery)
  - All numeric ops on CPU (uint8/int32 only). NO float intermediate states in the codec path.
  - Use Python stdlib (struct, zlib) plus numpy plus torch only. No new third-party deps.
  - Header is little-endian, 4-byte magic 'STCB', 4 x uint16 fields (N, H, W, num_classes), 4-byte uint32 boundary_count, 1-byte uint8 tile_size_log2.
  - Add a top-level docstring referencing Filler/Pevny/Fridrich 2011 STC paper and noting this is an STC-INSPIRED encoding (true syndrome trellis is overkill for v1 — we use boundary-restricted arithmetic coding which captures 80 percent of the wins per the design doc).
