# changes from upstream

FEC6 selector bolt-on around the public HNeRV lineage. The HNeRV decoder in `src/model.py` originates in PR [#95](https://github.com/commaai/comma_video_compression_challenge/pull/95) by `@AaronLeslie138` (`hnerv_muon`) and is byte-identical here. The byte substrate is PR [#101](https://github.com/commaai/comma_video_compression_challenge/pull/101) by `@SajayR` (`hnerv_ft_microcodec`), inheriting its compact decoder, latent-payload parsing, Brotli-coded source streams, and canonical Huffman for the latent sidecar. Prior lineage: PR [#98](https://github.com/commaai/comma_video_compression_challenge/pull/98) (`@EthanYangTW`), PR [#100](https://github.com/commaai/comma_video_compression_challenge/pull/100) (`@BradyMeighan`), PR [#102](https://github.com/commaai/comma_video_compression_challenge/pull/102), PR [#103](https://github.com/commaai/comma_video_compression_challenge/pull/103) (`@rem2`). This packet does not import PR #103's `constriction` range coder.

New in this packet: a 31-mode per-frame transform selector (K=16 active palette) chosen offline against the upstream scorer + a fixed-Huffman k=16 codebook over the selector indices. Selector framework is ~265 LOC of Python (`src/frame_selector.py`). Member `x` wraps the PR #101 source payload as `FP11 + source_len + source_pr101_payload + selector_len + selector_payload`; PR #101's Brotli envelope is unchanged and operates only inside the source-payload region, not at the ZIP layer and not over the appended FEC6 selector bitstream. Archive byte delta vs PR #101 source is +259 bytes (rate cost `25 * 259 / 37545489 = 0.000172`).

**Where this sits.** This packet hits a within-HNeRV-family local floor. PR #95 / #98 / #100 / #101 / #102 / #103 / this packet cluster within ~0.0008 on the CPU axis. CPU `0.192051 [contest-CPU]` is `−0.000794` vs PR #101 GOLD at `0.192845` (already includes the +259-byte rate cost); paired CUDA `0.226210 [contest-CUDA T4]`. For the tested frontier/medalist HNeRV-family archives, post-hoc general-purpose recompression of the actual contest `archive.zip` member bytes has no deliverable rate-term gain. To lower the contest-score frontier from here, the path likely runs through the full compression stack: pretraining and data priors; training objectives, curriculum, and scorer-aware search; architecture/substrate changes; quantization and post-training adaptation; entropy coding and archive grammar; decoder-side side information; and principled composition across those layers, not post-hoc recompression of the same member bytes. For maintainers and readers who want the broader research context: class/substrate-shift research lives in `comma-lab`: [`active candidate inventory`](https://github.com/adpena/comma-lab/blob/b7f16a081ee381803dd5d917bdaf805453fb81f3/docs/asymptotic_floor_candidate_inventory.md) and [`full-stack source map`](https://github.com/adpena/comma-lab/blob/b7f16a081ee381803dd5d917bdaf805453fb81f3/docs/full_stack_source_map.md), and reusable research tooling lives in [`adpena/tac`](https://github.com/adpena/tac).
