# changes from upstream

This submission stacks four small, separately reviewable changes onto the public HNeRV substrate established by @SajayR (PR [#95](https://github.com/commaai/comma_video_compression_challenge/pull/95) decoder) and the subsequent medal-class iterations by @AaronLeslie138 ([#98](https://github.com/commaai/comma_video_compression_challenge/pull/98)), @EthanYangTW ([#100](https://github.com/commaai/comma_video_compression_challenge/pull/100), [#102](https://github.com/commaai/comma_video_compression_challenge/pull/102)), @BradyMeighan ([#101](https://github.com/commaai/comma_video_compression_challenge/pull/101)), and @rem2 ([#103](https://github.com/commaai/comma_video_compression_challenge/pull/103)). The HNeRV decoder is byte-identical to @SajayR's reference implementation. We added:

1. **Frame Exploit Selector v6 (FEC6)** — a 31-mode deterministic per-frame transform space (identity, shifts, mirrors, rotations, channel rolls, gamma curves, scorer-targeted nudges) that decides at archive-build time which transform best preserves the upstream scorer's mask and pose response on each decoded frame. Selector indices are entropy-coded once into `0.bin` and replayed at inflate time.
2. **Fixed Huffman k=16** — a static 16-symbol Huffman table sized to the empirical selector-index distribution, replacing per-archive adaptive coding. The fixed table removes side-channel bytes and was independently the cheapest entropy choice across our sweep.
3. **Brotli quality 11 outer** — the standard outer compressor at maximum quality on the concatenated state-dict + latent + selector stream.
4. **Arithmetic-coded latent residuals** — a small arithmetic stream over residual nibbles of the latent payload, layered inside the brotli envelope.

Headline score against the current top merged submission ([@SajayR #101](https://github.com/commaai/comma_video_compression_challenge/pull/101), GOLD, `0.192845 [contest-CPU]`):

- `0.192051 [contest-CPU]` on Linux x86_64 via `upstream/evaluate.py --device cpu` — a `-0.000794` improvement on the axis the leaderboard ranks.
- `0.226210 [contest-CUDA T4]` on Modal T4 against the same archive bytes and runtime tree (canonical paired Modal A100 auth_eval; Modal call_id `fc-01KRMP4ZM5J8P1H3R2JN96VSC5`, 600 samples, 2026-05-15).

The CPU and CUDA values are reported as separate observations on 1:1 contest-compliant hardware per the standing dual-axis discipline. We do not extrapolate the mechanism behind the split; both numbers are presented as-measured.

## Limitations

- Single-video, contest-runtime target. FEC6 is selected per-frame against the upstream scorer on the contest video; we do not claim generalization to unseen dashcam clips.
- We have not used the four-day-shutdown clause; the submission is competitive on the CPU axis against the current top, and we believe also innovative under the 2026-05-11 new-submission gate ([@YassineYousfi #108 closure](https://github.com/commaai/comma_video_compression_challenge/pull/108)).
- The `report.txt` preamble embeds an absolute path string. This is the verbatim format `upstream/evaluate.py` writes; it does not affect any scored value and is left as-emitted for parity with prior medal-class submissions.
