# Structural Information Not Shipped

This memo canonicalizes the operator hypothesis:

> rate attacks + cooperative receiver + deterministic optimizer are one
> structural meta-paradigm: the decoder reconstructs useful structure from
> shared priors, deterministic constraints, already-charged bytes, or scorer
> invariants instead of shipping that structure as archive payload.

No score is claimed here. This is a research-only control memo and task
extractor. It must feed byte-closed prototypes, exact source/candidate replay,
and dual-axis custody before any promotion language is valid.

## Source Facts Re-derived

- `reports/latest.md` establishes a bifurcated current frontier: the public
  leaderboard reproduction axis and the internal CUDA promotion/regression
  axis are separate and currently have different champion families. This memo
  intentionally does not restate score values.
- `PROGRAM.md` and `upstream/evaluate.py` define the evaluator action as
  `100 * d_seg + sqrt(10 * d_pose) + 25 * rate`, where rate is charged archive
  bytes over source bytes.
- `upstream/modules.py` defines PoseNet as `FastViT-T12 -> vision(2048) ->
  summarizer(512) -> Hydra -> pose(12)`, while PoseNet distortion uses only
  `pose[..., : h.out // 2]`, i.e. six of twelve output dimensions.
- `upstream/modules.py` defines SegNet distortion as argmax-label
  disagreement, not full-logit distance.
- CPU/CUDA xray memos establish axis drift as diagnostic, not score authority:
  loader path, scorer-forward kernels, threshold geometry, and mixed coupling
  must be separated before any mechanism claim is valid.
- Cooperative-receiver/Wyner-Ziv memos establish the compliance boundary:
  compress-side scorer use can be allowed, but inflate-time scorer loading,
  external side channels, and uncharged hidden state are forbidden.
- Master-gradient/xray memos establish an advisory-rank prior: sensitivity is
  concentrated and rank-degenerate on at least one anchor, but archive-family
  generalization remains unproven without grammar-aware projection.

## Unified Action Formulation

Let:

- `theta`: emitted pixels / renderer parameters / archive grammar payload.
- `c`: codec and derivation configuration.
- `h`: hardware/evaluator axis.
- `a`: archive/runtime packet.
- `B_ship(a)`: bytes charged by the contest archive.
- `B_derive(theta,c,h)`: bytes or decisions deterministically derived by the
  receiver from public inputs, already-charged bytes, local computation, or
  scorer-invariant slack.
- `D_seg`, `D_pose`: exact evaluator distortions on the named axis.

Structural Information Not Shipped minimizes:

```text
S_total(theta,c,h,a) =
  100 * D_seg(theta,h)
  + sqrt(10 * D_pose(theta,h))
  + 25 * B_ship(a) / B_source
  + P_compliance(a,c,h)
  + P_axis_label(h)
  + P_runtime(a,c,h)
  - R_derived(B_derive_proven(theta,c,h))
```

`R_derived` is not a leaderboard term. It is an internal planner reward for
derived structure that has a byte-consumption proof. It is capped at zero if
the derivation requires an inflate-time scorer, hidden sidecar, network
download, untracked runtime dependency, axis coercion, or unverifiable
archive grammar.

The key move is to replace "ship a table / latent / model / selector" with
"derive it from constraints." The burden of proof shifts from plausibility to
three receipts:

1. The decoder consumes the derived structure.
2. The bytes needed to derive it are charged, public, or runtime-code-only
   within contest rules.
3. Same-runtime source/candidate replay preserves the claimed evidence axis.

## Free Compute, Free Bytes, And Scorer-Invariant Degrees Of Freedom

These are distinct and should never be collapsed:

| Surface | Definition | Legal use | Failure mode |
|---|---|---|---|
| Free compute | Search, scorer analysis, optimization, distillation, and fitting done before packaging, plus allowed inflate-time computation under runtime budget. | Spend compute to discover a smaller archive or deterministic decode. | Treating compress-side scorer state as available at inflate time. |
| Free bytes | Decoder-reconstructible structure from public inputs, already-shipped bytes, or code constants that are not archive payload. | Procedural codebooks, WZ side-info derivation, hash-seeded transforms, deterministic optimizers. | Hiding payload in source, constants, dependency wheels, filenames, external downloads, or untracked runtime. |
| Scorer-invariant degrees of freedom | Output changes that preserve SegNet argmax and/or PoseNet scored dimensions on the target axis. | Move bytes within null-space or slack space to reduce rate or improve the other axis. | Assuming invariance from tensor parity, advisory CPU runs, or one anchor without exact replay. |

## Hydra / Dual-Head Scorer-Architecture Exploit Analysis

The scorer has a structural asymmetry: it computes more than it scores.
PoseNet emits twelve pose-head values but the distortion only compares the
first six. SegNet emits full logits but the distortion only compares argmax
labels. These are not loopholes for inflate-time scorer dependence; they are
compress-time priors and constraint surfaces.

Actionable implications:

- F1 is real but bounded: dims 7-12 are unscored in the current source, so
  they can be used as auxiliary training targets, regularizers, or feature
  probes. They cannot be shipped as a hidden channel and cannot justify a
  score claim without exact replay of the first six dimensions.
- F2 is the SegNet analog: non-argmax logits are slack unless they cross the
  class boundary. The useful object is a margin certificate, not full-logit
  reconstruction.
- F3/F4/F5 are xray surfaces: vision(2048), summary(512), and Hydra ResBlock
  outputs expose where PoseNet is sensitive. Their legal product is a smaller
  generated packet, sensitivity map, or training curriculum, not scorer state
  inside inflate.
- F6/F7 are architecture guidance: if the scorer itself is trunk-plus-head,
  and PR95 used dual RGB heads, then a shippable decoder should prefer shared
  trunk / tiny axis-head / tiny selector designs over two full renderers.

The top Hydra probe should therefore be:

```text
source archive + runtime
  -> exact scorer xray at vision/summary/Hydra/pre-head/scored-six
  -> distilled tiny head schedule or null-space mask
  -> byte-closed archive export
  -> same-runtime scored-six + SegNet argmax replay
```

Any result that stops at internal feature parity is parser-consumption
evidence only. It is not full-frame inflate parity and not score authority.

## CPU-vs-GPU Structural Asymmetry Analysis

CPU/CUDA asymmetry is a structural-information channel only if axis labels are
honest. The public leaderboard CPU axis, internal CUDA promotion axis, macOS
CPU advisory axis, MPS, and proxy optimizer substrates are separate evidence
spaces.

Exploitation is compliant when:

- The packet declares its intended axis and runtime device contract.
- `[contest-CPU]` evidence is Linux x86_64, contest-compliant, same archive,
  same runtime, and full custody.
- `[contest-CUDA]` evidence uses the canonical CUDA evaluator path.
- Device-specific decode choices are encoded as explicit runtime behavior,
  not accidental auto-detect side effects.
- CPU/CUDA paired analysis records archive SHA, runtime content SHA, inflate
  device, evaluate device, raw-output aggregate hash when available, and
  PoseNet/SegNet deltas.

Exploitation is noncompliant or non-promotional when:

- CPU evidence is promoted as CUDA, or CUDA evidence is promoted as CPU.
- macOS CPU/MPS/MLX/Kaggle/proxy outputs are used to rank or kill exact
  contest candidates.
- The decoder branches on hidden environment state without packet metadata.
- An axis gap is called a method win before same-runtime replay and component
  recomputation are checked.

G2-G5 should be treated as disambiguators first, optimizers second. The first
task is to determine whether the axis effect is loader-dominated,
scorer-forward-dominated, threshold-geometry-dominated, or mixed. Only then
should SIMD packing, MKL/fp behavior, or cache/SM layout become an allocator
feature.

## Compliance Boundaries

- Inflate must remain scorer-free. No SegNet/PoseNet import, scorer weights,
  scorer feature extraction, or scorer-derived hidden state at inflate time.
- Compress-side scorer use is allowed only as a design/search/training prior
  that emits a byte-closed archive/runtime packet.
- Runtime code is not a payload landfill. Large constants, embedded blobs,
  dependency wheels, and source-encoded replay data require public-supplement
  hygiene and may still be disallowed.
- Seed expansion is not automatically free. Seed bytes are charged unless the
  seed is deterministically derived from public inputs or already-charged
  bytes. Generator code must be deterministic and audited.
- CPU/CUDA axis honesty is mandatory. Use `[contest-CPU]`,
  `[contest-CUDA]`, `[macOS-CPU advisory]`, and other proxy labels exactly.
- Tensor parity, latent parity, and scorer-feature parity are not full-frame
  inflate parity. Full-frame raw output or exact same-runtime eval is required
  for promotion language.
- Dykstra feasibility, posterior bands, and analytical action scores are
  planning tools. They do not replace archive bytes, runtime custody, and
  exact eval artifacts.

## Closing Verdict

Structural Information Not Shipped is not a new lane; it is a meta-paradigm
that makes several existing lanes legible as one mechanism:

- cooperative receiver: decoder has a shared prior;
- deterministic optimizer: optimal config is computable from constraints;
- rate attacks: charged bytes can be derived, collapsed, or avoided;
- Hydra/dual-head: scorer computes more structure than it scores;
- CPU/CUDA asymmetry: axis/runtime behavior is structure, not noise, when
  labeled and verified honestly.

The next valid Codex action is not more global theory. It is one byte-closed
task block above, preferably `structural_info_null_seed_codebook_p0` or
`hydra_slack_feature_xray_p0`, because both can turn this meta-paradigm into
a concrete artifact without mutating `.omx/state` or relying on a score claim.
