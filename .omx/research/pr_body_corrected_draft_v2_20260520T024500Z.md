The `x` payload is structured as `FP11 + source_len + source_pr101_payload + selector_len + selector_payload`:

1. **`FP11` magic + length prefixes** (local FEC6 wrapper grammar, not inherited from PR #101).
2. **PR #101 source payload** (`source_pr101_payload`, 178,158 bytes): contains the HNeRV state-dict at FP11 + the latent sidecar, both inside PR #101's Brotli (RFC 7932) envelope. Parsed by `src/codec.py` + `src/codec_sidecar.py` (inherited from PR #101).
3. **FEC6 selector payload** (`selector_payload`, 249 bytes = 6-byte header + 243-byte fixed-Huffman bitstream; **NEW**, not in PR #101). 600 selector indices, packed via the fixed-Huffman-k=16 table (243-byte bitstream is 1,944 bits = 3.24 bits/pair; full 249-byte wire payload is 3.32 bits/pair). The FEC6 selector is byte-appended to the source payload and is NOT further Brotli-compressed; the ZIP member `x` itself is stored uncompressed.

Section offsets and lengths are declared in the wrapper headers; the parser in `inflate.py` does not depend on any out-of-archive manifest.
