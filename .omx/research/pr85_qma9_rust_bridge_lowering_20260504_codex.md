# PR85 QMA9 Rust Bridge Lowering - 2026-05-04

## Scope

Lowered the QMA9 semantic mask decode/validation path from Python-only tooling
into a deterministic `runtime-rs` CLI surface. No remote GPU, training, or
exact-eval dispatch was performed. This is byte/parity evidence only and makes
no score claim.

## Changed Surface

- `runtime-rs/crates/qma-codec/src/lib.rs`
  - Hardened the adaptive QMA9 runtime model by replacing per-context heap
    `Vec<u32>` frequency tables with fixed arrays (`[u32; 2]`, `[u32; 5]`).
  - Existing Rust decode semantics and Python-contract fixture remain
    unchanged.
- `runtime-rs/crates/qma-codec/src/main.rs`
  - Added `qma-codec decode <input.qma9> <output.raw>`.
  - Added optional `--prefix-frames`, full shape checks, and metadata JSON.
  - Added `qma-codec metadata <input.qma9>` for fail-closed header validation.
- `runtime-rs/crates/qma-codec/tests/cli.rs`
  - Added Rust CLI fixture coverage using the embedded Python-contract QMA9
    payload and raw bytes.
- `src/tac/tests/test_qma9_rust_bridge.py`
  - Added Python bridge tests that generate QMA9 bytes with
    `encode_qma9_mask`, decode through the native Rust CLI, and assert
    byte-for-byte raw parity and prefix parity.

## Fixtures And Evidence

- Synthetic deterministic Python-reference fixture:
  - `frame_count=4`, `width=6`, `height=7`
  - raw bytes generated by `(t * 3 + y * 2 + x) % 5`
  - Rust CLI output matched Python raw bytes exactly.
- Prefix fixture:
  - `frame_count=3`, `width=5`, `height=4`
  - native `--prefix-frames 2` output matched the Python raw prefix exactly.
- Embedded Rust fixture:
  - `frame_count=2`, `width=3`, `height=4`
  - 37-byte QMA9 payload decoded to the 24-byte Python-contract raw fixture.

## Remaining Blockers

- The Rust QMA9 path now has deterministic CLI parity on Python-generated
  fixtures, but full PR85 public-payload parity is still bounded by local
  fixture availability and wall-clock cost for the 117,964,800-byte decoded
  mask.
- The existing STBM1BR Rust path was not changed in this pass; this work
  focused on the QMA9 validation/decode surface already present under
  `runtime-rs`.
