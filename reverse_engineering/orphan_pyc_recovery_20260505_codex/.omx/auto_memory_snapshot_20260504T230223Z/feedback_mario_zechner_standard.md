The quality bar for this codebase is Mario Zechner (libGDX creator).

**Why:** Mario Zechner built libGDX — a framework used by millions, beloved for its clean API design, thoughtful abstractions, and engineering that feels effortless because it's so well-crafted. That's the standard. Not "good enough to ship" — good enough that other engineers study it to learn how to write better code.

**How to apply:**
- Every public API should feel inevitable — like there's no other way it could have been designed
- Abstractions earn their existence through genuine reuse and clarity, not speculation
- Code should be readable as prose — a stranger opens any file and understands the intent in 30 seconds
- No clever tricks that require comments to explain — if it needs a comment, simplify the code
- Consistent patterns throughout — once you learn how one module works, you know how they all work
- Error handling that's helpful, not defensive — clear messages that point to the fix
- Names that are precise, domain-appropriate, and self-documenting
- The test of quality: would this code survive a public code review by senior engineers at a top company?
