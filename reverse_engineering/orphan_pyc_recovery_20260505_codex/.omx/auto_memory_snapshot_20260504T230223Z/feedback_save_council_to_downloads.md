Every time a council session returns with findings:

1. Save the FULL, DETAILED output to ~/Downloads/ as a markdown file
2. Name format: `council_TOPIC_YYYYMMDD.md`
3. Include ALL details, not summaries — the full deliberation
4. This is in ADDITION to saving to memory (which is summarized)
5. The Downloads folder becomes the permanent, unsummarized archive

**Why:** The user identified that summarization causes signal loss.
45 dropped signals were found in one audit. The full council output
contains nuances that get lost when I summarize for the conversation.

**How to apply:** After EVERY council/research agent completes:
1. Read the full output
2. Save complete output to ~/Downloads/council_*.md
3. THEN summarize for the conversation
4. THEN save key points to memory
